About Deadlock

What is a Deadlock?

A deadlock is a state in a multi-process system where two or more processes are unable to proceed because each is waiting for the other to release a resource. It’s like two people holding one end of two pencils each and waiting for the other to let go — and no one ever does.

Figure: Circular wait condition representing a Deadlock

Deadlocks typically occur when the following four conditions hold simultaneously:

  • Mutual Exclusion: Resources cannot be shared.
  • Hold and Wait: A process holds one resource while waiting for others.
  • No Preemption: Resources can’t be forcibly taken away.
  • Circular Wait: A cycle of processes waiting on each other exists.

Deadlock Avoidance

One strategy to avoid deadlocks is to ensure the system never enters an unsafe state. This is where the Banker's Algorithm comes in — it acts like a safety checker before granting resources.

About Banker's Algorithm

What is the Banker's Algorithm?

The Banker's Algorithm is a deadlock avoidance method named after a banker's approach to approving loans: the banker only grants a loan (resource) if they can guarantee that every customer (process) can be safely satisfied eventually.

It ensures the system always remains in a safe state — meaning there exists some sequence in which all processes can finish without causing a deadlock.

Key Concepts

  • Allocation: Current resources allocated to each process.
  • Max: Maximum resources each process may need.
  • Available: Resources currently available in the system.
  • Need: Calculated as Need[i][j] = Max[i][j] - Allocation[i][j].

How It Works

  1. A process requests resources.
  2. The algorithm checks if granting the request leads to a safe state.
  3. If safe, the resources are granted. Otherwise, the request is postponed.

Figure: Banker's Algorithm Decision Flow

This way, the system avoids unsafe states that might lead to deadlocks while still ensuring fair resource utilization.
