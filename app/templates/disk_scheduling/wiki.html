Disk Scheduling

Overview

Disk scheduling is done by operating systems to schedule I/O requests arriving for the disk. Disk scheduling is also known as I/O scheduling. It is crucial for optimizing the performance of the system because:

  • Multiple I/O requests may arrive from different processes, and only one I/O request can be served at a time by the disk controller. Thus, other I/O requests need to wait in the waiting queue and need to be scheduled.
  • Two or more requests may be far from each other, which could result in greater disk arm movement.
  • Hard drives are one of the slowest parts of the computer system, so efficient disk access is necessary.

Important Terms in Disk Scheduling

Before diving into the disk scheduling algorithms, letâ€™s have a quick look at some important terms:

  • Seek Time: Seek time is the time taken for the disk arm to locate a specified track where the data is to be read or written. The disk scheduling algorithm that minimizes the average seek time is preferable.
  • Rotational Latency: Rotational latency is the time taken for the desired disk sector to rotate into position so that the read/write heads can access it. A scheduling algorithm that minimizes rotational latency is better.
  • Transfer Time: Transfer time refers to the time required to transfer data. It depends on the disk's rotational speed and the number of bytes to be transferred.
